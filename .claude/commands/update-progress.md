# 進捗（progress）更新スキル

このスキルは、実装タスクの完了を記録するために使用します。

## 手順

1. `progress.txt` を読み込む
2. ユーザーに完了したタスクを確認する
3. 完了したタスクに `[x]` を付ける
4. 完了日時を記録する
5. 「最終更新」日付を今日の日付に更新する
6. テスト結果があれば更新する

## 実行フロー

### Step 1: ファイル読み込み
まず `progress.txt` を読み込み、現在の状態を確認してください。

### Step 2: 完了タスクの確認
AskUserQuestion ツールを使って、以下のPhaseから完了したタスクを確認してください：

**Phase一覧:**
- Phase 1: 基盤・設定
- Phase 2: 認証・OAuth連携
- Phase 3: コア機能（バックエンド）
- Phase 4: UI画面
- Phase 5: UI連携・統合
- Phase 6: Firestore連携
- Phase 7: 通知機能
- Phase 8: 自動実行（Cloud Functions）
- Phase 9: サブスクリプション
- Phase 10: テスト・品質

具体的にどのタスク（例: 6.1, 8.2など）が完了したか確認してください。

### Step 3: ファイル更新
- `[ ]` を `[x]` に変更
- 完了日時を追加（例: `[x] 6.1 DailyLog保存機能  完了: 2026-01-24`）
- 「最終更新」の日付を今日に更新

### Step 4: テスト結果更新
テストを実行した場合は、「テスト実行結果」セクションを更新してください：
- Test Suitesの数
- Testsの総数
- 各テストファイルの結果

### Step 5: 関連タスク確認
`manual-work.txt` に関連する外部作業がある場合は、そちらの更新も提案してください。

### Step 6: 確認報告
更新した内容をユーザーに報告してください。

## 注意事項

- 必ずユーザーに確認してから更新すること
- テストが成功していないタスクは完了にしないこと
- 型チェック（`npx tsc --noEmit`）が通っていることを確認すること
- 完了日時は必ず記録すること
