# 手動作業（manual-work）更新スキル

このスキルは、外部サービス設定作業の完了を記録するために使用します。

## 手順

1. `manual-work.txt` を読み込む
2. ユーザーに完了した作業を確認する
3. 完了した項目に `[x]` を付ける
4. 完了日時を記録する
5. 「最終更新」日付を今日の日付に更新する
6. 関連する `progress.txt` のタスクがあれば更新する

## 実行フロー

### Step 1: ファイル読み込み
まず `manual-work.txt` を読み込み、現在の状態を確認してください。

### Step 2: 完了項目の確認
AskUserQuestion ツールを使って、以下のカテゴリから完了した作業を確認してください：

**カテゴリ一覧:**
- セクション1: レシート検証シークレット設定
- セクション2: App Store Connect設定
- セクション3: Firebase設定
- セクション4: OAuth認証情報設定
- セクション5: Expo/EAS設定
- セクション6: 環境変数設定
- セクション7: 本番リリース前の作業

具体的にどの項目（例: 2.1, 2.2, 3.1など）が完了したか確認してください。

### Step 3: ファイル更新
- `□` を `[x]` に変更
- 完了日時を追加（例: `[x] 2.1 Apple Developer Program登録  ← 完了: 2026-01-24`）
- 「最終更新」の日付を今日に更新

### Step 4: progress.txt 確認
関連するタスクが `progress.txt` にある場合は、そちらも更新してください。

### Step 5: 確認報告
更新した内容をユーザーに報告してください。

## 注意事項

- 必ずユーザーに確認してから更新すること
- 完了していない項目を誤って完了にしないこと
- 完了日時は必ず記録すること
